这是一个基于 Tauri 的桌面学习应用。

核心体验：
- 右边栏展示 PDF。
- 左边栏与 LLM 交互，体验接近 Claude Code，但面向学习场景。
- 用户点击右侧 PDF 的某一页后，应用将该页渲染为图片，并交给默认支持多模态的 LLM。
- LLM 主动开始交互式学习：讲解知识点、中途提问、评估用户回答、讲完一页后进行小测试。
- 用户可以随时打断 LLM，提出问题，之后回到当前学习流程。

技术栈：
- 桌面壳：Tauri
- 前端：Vue + Vite + TypeScript
- 后端：Rust
- PDF 页输入：第一版直接支持图片输入，默认 LLM 具备多模态能力
- 笔记/导出：Typst

第一版范围：
- 支持打开 PDF。
- 支持在右侧渲染 PDF 页面。
- 点击某页后，将当前页渲染图作为主输入发送给多模态 LLM。
- 如果能抽取文本，则将页文本作为辅助输入，但第一版不依赖 PDF 文本抽取。
- 左侧支持流式 LLM 输出。
- 支持用户中断当前讲解并提问。
- 支持基础状态机，驱动讲解、提问、等待回答、评价、总结等流程。
- 支持每页学习记录的基础结构，为后续 Typst 笔记生成做准备。

暂不作为第一版重点：
- 不做完整 PDF 文本解析流水线。
- 不强依赖 OCR。
- 不把 Typst 放进实时聊天渲染链路。
- 不让 LLM 自由生成完整 Typst 文档。

Typst 策略：
- Typst 是学习笔记、页面总结、错题本和导出 PDF 的目标格式。
- LLM 不直接写完整 Typst 文档，而是输出结构化学习内容。
- 应用使用固定 Typst 模板生成稳定文档。
- 如果 Typst 编译失败，可以将错误和结构化数据交给 LLM 做受限修复。

左侧交互：
- 使用 Vim-inspired 的键盘优先体验。
- 不要求完整复刻 Vim。
- 需要支持普通模式、输入模式、中断、搜索、命令面板、消息选择和页面跳转等基础操作。
